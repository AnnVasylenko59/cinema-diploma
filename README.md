# Інформаційна вебсистема онлайн-кіноафіші з бронюванням квитків, інтерактивною мапою сеансів та персоналізованою системою рекомендацій

## 📋 Про проєкт
Дана система розроблена як комплексна вебплатформа для моніторингу кіносеансів, взаємодії з інтерактивною картою кінотеатрів України, автоматизації процесу бронювання місць та отримання персоналізованих рекомендацій.

Проєкт реалізований як **Monorepo**, що об'єднує клієнтську та серверну частини для забезпечення цілісності розробки та розгортання.

## 📂 Структура репозиторію

| Папка / Файл | Опис |
| :--- | :--- |
| `backend/` | Серверна частина застосунку (Node.js, Express, Prisma) |
| `frontend/` | Клієнтська частина застосунку (React, Vite, Tailwind CSS) |
| `.github/` | Налаштування CI/CD (GitHub Actions для автоматичних тестів) |
| `.gitignore` | Уніфікований файл ігнорування системних файлів |
| `LICENSE` | Ліцензія на використання коду (MIT) |

## 🛠 Технологічний стек

### Backend:
- **Runtime:** Node.js (v18+)
- **Framework:** Express.js
- **Database:** Prisma ORM (MySQL/PostgreSQL)
- **Validation & Auth:** JWT, custom middleware
- **Testing:** Jest, Supertest

### Frontend:
- **Library:** React (Vite)
- **Styling:** Tailwind CSS
- **Icons & UI:** Lucide React, Framer Motion
- **Maps:** Leaflet / Google Maps API integration

## 🌟 Ключові функції

### 🎬 Для користувачів:
- **Інтерактивна мапа:** Пошук кінотеатрів та перегляд актуальних сеансів у різних містах України з використанням геоданих.
- **Система бронювання:** Вибір конкретних місць у залі в реальному часі та **автоматична генерація електронних квитків у форматі PDF**.
- **Керування вподобаннями:** Система оцінювання фільмів, створення списків «Улюблене» (Watchlist) та доступ до детального архіву переглядів.
- **Персоналізація профілю:** Налаштування кабінету користувача (зміна нікнейму, аватара), підтримка **зміни теми сайту (Dark/Light mode)** та **повна локалізація інтерфейсу**.
- **Інтелектуальні рекомендації:** Алгоритм підбору контенту на основі історії переглядів та індивідуальних оцінок користувача.

### 🔐 Для адміністраторів:**
- **Управління контентом:** Додавання та редагування сеансів, керування базою фільмів та інформацією про акторів.
- **Цінова політика:** Гнучке налаштування вартості квитків залежно від часу сеансу чи типу залу.
- **Аналітика та статистика:** Панель моніторингу продажів та візуалізація статистики популярності фільмів для прийняття управлінських рішень.

## 🧪 CI/CD та тестування
У проєкті налаштована автоматизація через **GitHub Actions**. Файл `.github/workflows/api-tests.yml` забезпечує:

## ⚙️ Встановлення та запуск (Інструкція для розробників)

Нижче наведено покрокову інструкцію для розгортання проєкту на свіжовстановленій ОС.

### 0. Необхідні залежності та програмне забезпечення
Перед початком переконайтеся, що на вашому комп'ютері встановлено:
- **Git** — система контролю версій.
- **Node.js** (v18 або новіше) та **npm** — середовище виконання.
- **PostgreSQL** — система керування базами даних (необхідно створити локального користувача та пусту базу даних для проєкту).
- Редактор коду (рекомендовано VS Code або IntelliJ Idea).

### 1. Клонування проєкту:
```bash
git clone https://github.com/AnnVasylenko59/cinema-diploma.git
cd cinema-diploma
```

### 2. Налаштування середовища, бази даних та Backend

Для підключення до бази даних та налаштування змінних середовища виконайте наступні кроки:

#### Backend

```bash
cd backend
npm install
```

1. Створіть файл `.env` на основі `.env.example`.
2. Вкажіть у ньому змінну `DATABASE_URL` у форматі:

```text
postgresql://ВАШ_ЮЗЕР:ВАШ_ПАРОЛЬ@localhost:5432/НАЗВА_БАЗИ?schema=public
```

3. Створіть таблиці у базі даних (застосуйте міграції):

```bash
npx prisma migrate dev
```

4. Запустіть проєкт у режимі розробки:

```bash
npm run dev
```

---

### 3. Налаштування Frontend

Відкрийте нову вкладку терміналу та перейдіть у папку фронтенду:

```bash
cd ../frontend
npm install
npm run dev
```

---

### 4. Базові команди та операції

- `npm run dev` — запуск застосунку в режимі розробника (з гарячим перезавантаженням).
- `npx prisma studio` (у папці `backend`) — запуск графічного вебінтерфейсу для перегляду та керування даними в базі даних.
- `npx prisma migrate dev` (у папці `backend`) — створення та застосування нової міграції при зміні структури БД.
- `npm run build` (у папці `frontend`) — створення оптимізованої збірки фронтенду для продакшену.

## 📝 Стандарти документування коду

Для забезпечення підтримки проєкту використовується стандарт **JSDoc**.

### Основні правила:

- **Коментарі:** кожна публічна функція або компонент повинні мати блок `/** ... */`.
- **Параметри:** використовуйте `@param` для опису вхідних даних.
- **Результат:** використовуйте `@returns` для опису того, що повертає функція.
- **Асинхронність:** обовʼязково додавайте тег `@async` для промісів.

## Збір технічних даних та звітів про помилки
Для моніторингу стабільності клієнтської частини та збору розширених даних про помилки інтегровано платформу **Sentry** (`@sentry/react`).
* **Збір технічних даних:** Система автоматично збирає інформацію про середовище користувача (ОС, версія браузера, IP-адреса, роздільна здатність екрана).
* **Дампи стану (Скріншоти/Відео):** Використовується `replayIntegration` (Session Replay) для запису DOM-дерева та дій користувача перед виникненням помилки. Це дозволяє візуально відтворити проблему у вигляді відео-дампу.
* **Кроки відтворення:** Через `Sentry.showReportDialog()` реалізовано форму зворотного зв'язку. При натисканні на "Повідомити про проблему" користувач може детально описати свої дії, які автоматично прив'язуються до технічного звіту.
