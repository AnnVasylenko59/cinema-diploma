# Документація статичного аналізу коду (Linting)

## Обраний лінтер
Для проєкту «Кіноафіша» обрано **ESLint v10**.

## Причини вибору
1. **Універсальність:** Дозволяє перевіряти як React-компоненти фронтенду, так і Node.js API бекенду в межах одного Monorepo репозиторію.
2. **Екосистема плагінів:** Підтримка специфічних розширень для React та Node.js забезпечує дотримання кращих практик розробки та виявлення помилок на ранніх етапах.

## Базові правила
- **indent: 4** — встановлено як стандарт проєкту для забезпечення чіткої структури та читабельності коду.
- **quotes: single** — використання одинарних лапок для уніфікації рядкових літералів.
- **semi: always** — обов'язкова крапка з комою для запобігання неоднозначності при інтерпретації коду.
- **no-unused-vars** — автоматичне виявлення оголошених, але не використаних змінних для підтримки чистоти кодової бази.

## Ігнорування директорій
До списку ігнорування додано папку `backend/generated/`, оскільки вона містить автоматично згенерований код Prisma Client, який не підлягає ручному редагуванню.

## Інструкція з запуску
- Перевірка проєкту: `npx eslint .`
- Автоматичне виправлення: `npx eslint . --fix`
